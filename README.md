# 🚗 汽车之家用户口碑深度分析系统

基于 DeepSeek 大模型的智能化汽车口碑分析工具。本系统包含两大核心模块：
1. **多维度满意度分析**：提取 16 个产品维度的情感倾向与体验类型。
2. **本品与竞品对比分析**：自动识别竞品车型并判断优劣势。

## 🛠️ 环境配置

### 1. 环境要求
- Python 3.8+
- 建议使用 Anaconda 管理环境

### 2. 安装依赖库
```bash
pip install pandas openpyxl matplotlib seaborn openai tqdm
```

### 3. 配置 API Key
本系统使用 DeepSeek 大模型接口。请在系统环境变量中设置您的 API Key：
- **变量名**：`DEEPSEEK_API_KEY`
- **变量值**：您的 DeepSeek API Key 字符串

**Windows 设置方法**：
```cmd
setx DEEPSEEK_API_KEY "your-api-key-here"
```
*设置后请重启 IDE 或终端以使环境变量生效。*

## 📊 输入数据格式

两个脚本均依赖同一份数据源：`../10.汽车口碑数据/merged_autohome_reviews_V2.xlsx`。

该 Excel 文件需包含以下关键字段：
| 字段名 | 说明 |
| :--- | :--- |
| `车型名称` | 当前讨论的车型（本品） |
| `车型版本` | （竞品分析必需）具体的配置版本 |
| `最满意` | 用户评价中最满意的部分 |
| `最不满意` | 用户评价中最不满意的部分 |
| `空间评论`、`驾驶感受评论` 等 | 各维度的细分评论文本 |

*程序会自动将上述评论文本合并为一个完整的“合并评价”字段用于分析。*

## 🧩 模块一：满意度深度分析

**脚本文件**：`6.2 API进行满意度分析V3-合并分析.py`

### 功能特性
- **16 维情感分析**：覆盖品牌、安全、操控、动力、质量、外观、舒适、使用成本、购车价格、配置、空间、内饰、服务、权益、智能座舱、智能驾驶。
- **七档评分**：-2（很不满意）到 2（很满意），支持 0.5 精度。
- **体验类型区分**：判断提及点是 **购前即可感知** (`pre-purchase`) 还是 **长期使用后方能体会** (`long-term`)。
- **可视化报表**：自动生成情感分布图、热力图、评分箱线图等。

### 运行方式
```bash
python "6.2 API进行满意度分析V3-合并分析.py"
```

### 输出说明
- **分类结果明细.xlsx**：每一行是一条 API 返回的具体维度分析记录（含证据原文、评分）。
- **满意度分析结果.xlsx**：汇总统计表（整体统计、按车型统计、体验类型统计）。
- **PNG 图表**：包括整体情感分布、车型热力图、体验类型对比图等。

---

## 🆚 模块二：本品与竞品对比分析

**脚本文件**：`1.0 本品车和竞品车对比分析.py`

### 功能特性
- **智能竞品识别**：利用 DeepSeek **推理模型** (`deepseek-reasoner`) 自动从评论文本中提取用户对比过的竞品车型。
- **优劣势判定**：精准判断本品相对于竞品是 **更好 (better)**、**持平 (same)** 还是 **更差 (worse)**。
- **净优势分计算**：量化本品在各竞品面前的综合竞争力。
- **JSON 自动修复**：内置鲁棒的 JSON 修复算法，有效应对大模型输出截断问题。

### 运行方式
```bash
python "1.0 本品车和竞品车对比分析.py"
```

### 输出说明
- **竞品对比明细.xlsx**：每条记录包含对比车型、对比维度、结果、证据原文。
- **竞品对比分析报告.xlsx**：包含按竞品统计、按类别统计、TOP15 优劣势点等 Sheet。
- **竞品对比统计摘要.txt**：文字版分析摘要，方便快速查阅结论。
- **可视化图表**：
    - `1_主要竞品分析.png`：提及次数与净优势分双轴图。
    - `2_各类别对比结果.png`：堆叠条形图展示各维度优劣势分布。
    - `3_竞品对比热力图.png`：本品 vs 主要竞品的多维竞争力矩阵。
    - `4_优劣势TOP10.png`：本品最突出的优势与劣势组合。

## ⚠️ 注意事项

1. **API 费用**：两个脚本均会大量调用 DeepSeek API，请确保账户余额充足。
2. **并发控制**：代码中使用 `ThreadPoolExecutor` 多线程加速，可通过修改 `MAX_WORKERS` 变量调整并发数（默认为 10）。
3. **数据断点续跑**：
   - 竞品分析脚本支持检测已有结果文件。若 `竞品对比明细.xlsx` 已存在，将**跳过 API 调用直接进入统计绘图阶段**。
   - 满意度分析脚本每次运行均会重新调用 API，如需断点续跑请自行修改代码逻辑。
4. **车型版本字段**：竞品分析脚本强制要求数据表中包含 `车型版本` 列，若缺失会报错退出。
