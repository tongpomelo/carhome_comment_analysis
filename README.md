# 汽车口碑智能分析系统

基于 DeepSeek 大模型的汽车之家口碑数据自动化分析工具，包含两大核心模块：

- **满意度深度分析**：从多维度评论中提取用户对本品车的满意度评分、体验类型（购前/长期）及情感倾向。
- **竞品对比分析**：自动识别评论中提及的竞品车型，判断本品车相较于竞品是“更好/持平/更差”，并量化优劣势分布。

---

## ✨ 功能特点

### 1. 满意度分析 (`6.2 API进行满意度分析V3-合并分析.py`)

- 支持合并“最满意”“最不满意”“空间评论”等 **11 个独立评论字段**，形成完整评论文本。
- 调用 DeepSeek API 精准识别 **16 个汽车产品类别**（品牌、安全、操控、动力、质量、外观、舒适、使用成本、购车价格、配置、空间、内饰、服务、权益、智能座舱、智能驾驶）。
- 对每个类别输出：
  - 情感标签（满意/中立/不满意）
  - 强度评分（-2 ～ 2，支持 0.5 精度）
  - 原文证据
  - 目标车型归属（本车/其他车）
  - 体验类型（购前感知 `pre-purchase` / 长期体验 `long-term`）及分类理由
- 多线程并发调用，大幅提升处理效率。
- 自动生成：
  - 整体/分车型的类别提及频次、情感分布、平均评分统计表
  - 体验类型专项分析（分布、评分对比、可视化图表）
  - 9 张以上 PNG 图表（柱状图、堆叠图、热力图、箱线图、散点图等）
  - 完整的 Excel 报表（包含全量明细、统计汇总）

### 2. 竞品对比分析 (`1.0 本品车和竞品车对比分析.py`)

- 采用 **DeepSeek Reasoner 推理模型**，准确捕捉评论中的竞品对比关系。
- 自动提取对比信息：
  - 竞品车型标准名称
  - 对比类别（同上 16 个类别）
  - 对比结论（`better` 本品更优 / `same` 持平 / `worse` 本品更差）
  - 支持证据原文
- 统计输出：
  - 各竞品车型的提及次数、优/平/劣次数、净优势分、优势率
  - 各类别的对比结果分布
  - TOP15 优势点与劣势点明细
- 生成 4 张可视化图表（竞品分析组合图、类别对比堆叠图、优劣势热力图、优劣势 TOP10 条形图）
- 输出文字摘要报告，快速掌握本品竞争力概况。

## 🚀 快速开始

### 环境要求

- Python 3.8+
- 有效的 DeepSeek API Key

### 安装依赖

```bash
pip install pandas openpyxl matplotlib seaborn openai tqdm
```

### 配置 API Key

将你的 DeepSeek API Key 设置为系统环境变量：

**Windows (CMD):**
```cmd
set DEEPSEEK_API_KEY=your_api_key_here
```

**Windows (PowerShell):**
```powershell
$env:DEEPSEEK_API_KEY="your_api_key_here"
```

**macOS / Linux:**
```bash
export DEEPSEEK_API_KEY="your_api_key_here"
```

> 建议将上述命令添加到 `~/.bashrc` 或 `~/.zshrc` 中持久化。

### 准备输入数据

确保在 `../10.汽车口碑数据/` 目录下存在 Excel 文件 `merged_autohome_reviews_V2.xlsx`。

**必要列：**
- `车型名称`、`车型版本`
- 评论文本字段（至少包含以下之一）：
  - `最满意`、`最不满意`
  - `空间评论`、`驾驶感受评论`、`续航评论`
  - `外观评论`、`内饰评论`、`性价比评论`
  - `智能化评论`、`油耗评论`、`配置评论`

> 程序会自动合并这些字段为完整评论，缺失字段不影响运行。

### 运行分析

#### 1. 执行满意度分析

```bash
python "6.2 API进行满意度分析V3-合并分析.py"
```

程序将：
- 读取数据并合并评论
- 多线程调用 DeepSeek API 进行分析
- 在 `最满意API分析_V2/` 目录下生成结果

#### 2. 执行竞品对比分析

```bash
python "1.0 本品车和竞品车对比分析.py"
```

程序将：
- 读取同一份输入数据
- 使用推理模型提取竞品对比信息
- 在 `竞品对比分析结果/` 目录下生成报告

> 若已存在 `竞品对比明细.xlsx`，程序会跳过 API 调用，直接加载现有数据进行统计分析，方便重复绘图调参。

## ⚙️ 高级配置

可在程序开头修改以下变量以适应需求：

### 通用配置

| 变量            | 默认值 | 说明                              |
| :-------------- | :----- | :-------------------------------- |
| `MAX_WORKERS`   | 10     | 并发线程数，根据 API 速率限制调整 |
| `RETRY_TIMES`   | 3      | API 调用失败重试次数              |
| `REQUEST_DELAY` | 0.1    | 每次 API 请求后的间隔（秒）       |

### 满意度分析专属配置

| 变量          | 默认值                                                 | 说明                    |
| :------------ | :----------------------------------------------------- | :---------------------- |
| `OUTPUT_DIR`  | `"最满意API分析_V2"`                                   | 输出文件夹名称          |
| `INPUT_EXCEL` | `"../10.汽车口碑数据/merged_autohome_reviews_V2.xlsx"` | 输入文件路径            |
| `SHEET_NAME`  | `"merged_autohome_reviews"`                            | 输入 Excel 中的工作表名 |

### 竞品分析专属配置

| 变量         | 默认值                | 说明                           |
| :----------- | :-------------------- | :----------------------------- |
| `OUTPUT_DIR` | `"竞品对比分析结果"`  | 输出文件夹名称                 |
| `MODEL`      | `"deepseek-reasoner"` | 使用的模型（推荐保持推理模型） |

## ❓ 常见问题

### 1. API 调用失败或返回空结果？

- 检查网络连接及 API Key 是否有效。
- 若频繁触发限流，可适当调低 `MAX_WORKERS` 或增大 `REQUEST_DELAY`。
- 确认输入评论非空且包含有效文本。

### 2. 图表中文显示为方框？

程序已设置中文字体回退（`SimHei`, `Microsoft YaHei`）。若仍异常：
- Windows 用户通常无需额外操作。
- Linux/macOS 用户可安装中文字体或修改 `plt.rcParams['font.sans-serif']` 为系统已有字体。

### 3. 竞品分析中本车车型是如何确定的？

程序默认将输入数据中第一行的 `车型名称` 作为“本车车型”。若数据包含多款车型，建议先按车型拆分 Excel 再分别运行竞品分析，以保证对比逻辑正确。

### 4. 想只分析“本车”相关评论？

满意度分析程序已通过 `是否本车 == "this"` 过滤，竞品分析程序则默认所有对比均为本车视角（由提示词强制约束）。
